You’re running a client‑side React app. The browser can only fetch files it’s explicitly told about— it can’t “look inside” a folder at runtime.
To auto‑populate each filter with every image that lives in its matching folder, you need to feed React a list of those files at build‑time, not at runtime. There are two clean ways to do that:

Option	How it works	Pros	Cons
A. Move images into /src and let Vite build the list with import.meta.glob	Put assets in src/assets/haircuts/{folder}/….  One line of code globs everything and gives you an object whose keys are the paths.	• Zero manual lists• No backend• Works today with pure front‑end code	• You must relocate images out of public
B. Keep images in /public and run a tiny Node script that scans the folders and writes a JSON manifest (e.g. haircutImages.json) into /src	Script runs once (or on every build) and produces { "Fades": [ … ], … }.  React imports that JSON.	• You keep assets where they are	• Requires the extra script (one 15‑line file)

Pick one.
I recommend Option A—less moving parts, nothing extra to maintain, and it’s one prompt away.

2. Create a helper file at src/features/haircuts/useHaircutImages.js with:

   import { useMemo } from 'react';

   // Auto‑import every image in sub‑folders of assets/haircuts
   const modules = import.meta.glob('/src/assets/haircuts/*/*.{jpg,jpeg,png,webp}');

   // Build an object { "fades": [url1,url2], ... }
   export function useHaircutImages() {
     return useMemo(() => {
       const map = {};
       for (const path in modules) {
         const [, folder, file] = path.match(/haircuts\/([^/]+)\/([^/]+)$/);
         if (!map[folder]) map[folder] = [];
         map[folder].push(modules[path]);  // Vite turns this into the URL
       }
       return map;
     }, []);
   }
